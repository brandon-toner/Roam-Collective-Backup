                - For example, my large database, 9000+ pages and nearly 700 queries has a stutter. If I actively filter OUT `[query](<query.md>)` via a global filter, my database speeds up dramatically.
                    - Even as I write this, focused on just my blocks, there is a significant stutter which I don't see on ANY of my other graphs that I work on - and I have a brand new M1 mac that screams. Roam Portal, for example, is no issue rendering 100,000 blocks in the 3D visualizer.  
                - This is belief is not without arguments against it, given that some folks indicate little difference in performance between activating and deactivating the global filter. 
                    - However, Rob Haisfield and Viktor (just to name two) have quite significant database slowdowns. 
            - Take these opinions with some healthy skepticism, but I just thought I would mention it given the trend I see here. I have moved all my clients - especially the large teams - away from putting "temporary" queries up. It's one thing to have a dashboard page (such as on our directory pages) which house a set of queries. But it is another to have several on EVERY DNP for the next year.
